# Walkthrough - level07

The program has SUID permissions from the user `level08`, which will allow us to obtain their flag.

# Dissassembly

- `memset` à 0 des arguments du programme et des variables d'environnement
- Une boucle qui attend des commandes sur la stdin `store`, `read` et `quit`
- Un buffer de 400 bytes qui sert à stocker des `int` (`v6`)
- Des fonctions `read_number` et `store_number` qui utilise le buffer `v6`

# Exploit

L'objectif est d'écraser l'EIP pour exploiter un return-to-libc.

La fonction store_number stock un nombre à un index donné mais ne vérifie pas si l'index est bien dans le buffer (`v6`). Nous pouvons donc overflow le buffer pour trouver l'addresse de l'EIP. 

Pour cela, utilisons GDB:

Comme on peut le voir, l'addresse de l'index 40 est `0xffffd5f4`, donc l'adresse du début du buffer vaut:

```
40 * 4 = 0xA0
0xffffd5f4 - 0xA0  = 0xffffd554
```

Calculons l'offset entre le debut du buffer et l'EIP:

```
0xffffd71c - 0xffffd554 = 0x1C8
```

Soit 456 bytes, donc l'index de l'EIP est `456 / 4 = 114`

Comme 114 est un multiple de 3, le fonction `store_number` empeche de stocker un nombre à cet index. Nous allons donc exploiter un overflow de l'int pour pouvoir.

```
UINT_MAX / 4 + INDEX = NEW_INDEX
4294967296 / 4 + 114 = 1073741938
```
